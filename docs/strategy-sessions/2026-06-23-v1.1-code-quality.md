## Recommendation (Stage 6) & Decision (Stage 7)

**Recommended and chosen: Path B.** The `#1646` class is the correct target (user goal + the #1 empirical recurrence), but the structural cure — converging websocket.py's ~1000-line WS dispatch + ~16 WS-only behaviors onto `ViewRuntime` — is the single highest-blast-radius change in the codebase, which v1.0.8-1 deliberately soak-gated under `#1122`. Path B honors that discipline: ship the nets + DRY + hygiene + canon first (immediate, continuous quality payoff that builds the mechanical safety net), **then** land the convergence once drift is structurally detectable — exactly the `#1859` + `#1122` discipline that made the v1.0.8-1 convergence safe. Path A-first risks a giant un-netted PR; Path C-first reorganizes code the convergence will move again.

This **overrides the prior Path-E "defer headline"** decision — now justified by completed launch-soak data — so it is a directional change recorded in **ADR-022** (Proposed).
