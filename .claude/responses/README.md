# PR Review Responses

This directory contains responses to PR review feedback, documenting all actions taken to address reviewer comments.

```

    ├── response-{timestamp}.md     # Historical responses

```

## Usage

### Responding to Review Feedback

Use the `/respond` command to automatically address review feedback:

```bash
# Respond to latest review of PR #40
/respond 40
```

The agent will:
1. Read the latest review from `.claude/reviews/pr-40/latest.md`
2. Categorize all feedback items (address now / defer / acknowledge)
3. Make code changes for quick fixes
4. Create GitHub issues for deferred tasks
5. Answer reviewer questions
6. Save response document
7. Update response index

### Accessing Responses

**Latest Response**:
```bash
cat .claude/responses/pr-40/latest.md
```

**Specific Response**:
```bash
cat .claude/responses/pr-40/response-2025-01-12-180530.md
```

**All Responses for a PR**:
```bash
ls -la .claude/responses/pr-40/
```

## Response Categories

### ✅ Addressed (Code Changes Made)

Feedback items that were fixed immediately with code changes:
- Simple fixes (< 30 minutes)
- Clear implementation path
- Non-controversial changes
- Examples: Add docstring, fix typo, improve naming

### 📋 Deferred (GitHub Issues Created)

Feedback items that require more work and were tracked as issues:
- Large tasks (> 30 minutes)
- Require design discussion
- Need new infrastructure
- Examples: Add test suite, refactor architecture, new feature

### ❓ Acknowledged (Questions Answered)

Questions from reviewer that were answered:
- Explanations of design decisions
- Rationale for approach taken
- Context for implementation choices

## Response Quality Standards

All responses follow these standards:
- ✅ All feedback items addressed (fixed, deferred, or answered)
- ✅ Code changes tested and verified
- ✅ GitHub issues created for deferred items
- ✅ Clear explanations for questions
- ✅ Next steps documented

## Workflow

### Typical PR Review Response Workflow

1. **Review is completed**: `/review-save 40`
   - Review saved to `.claude/reviews/pr-40/review-{timestamp}.md`

2. **Address feedback**: `/respond 40`
   - Agent reads review
   - Makes code changes
   - Creates issues for deferred items
   - Saves response to `.claude/responses/pr-40/response-{timestamp}.md`

3. **Commit changes**:
   ```bash
   git add .
   git commit -m "fix(review): Address all feedback from PR #40 review"
   git push
   ```

4. **Update PR**:
   - New commits address review feedback
   - Reference response document in PR comments
   - Link to created issues

5. **Ready for re-review** or **merge**
