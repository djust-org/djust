## What This PR Does

### 1. Client.js Refactoring (Bug Fix + Code Quality)

**Problem Solved:**
- Duplicate response handling logic existed in both WebSocket.handleMessage() and HTTP fallback paths
- Inconsistent behavior: optimistic state cleanup was missing from WebSocket path
- Critical bug: `optimistic-pending` CSS class not removed, causing stuck loading cursor on @optimistic toggles
- Maintenance burden of keeping two implementations in sync

**Solution:**
- Created centralized `handleServerResponse(data, eventName, triggerElement)` function
- Both WebSocket and HTTP fallback now use the same handler
- Removed duplicate `applyPatches()` implementation from WebSocket class
- Eliminated 8 patch helper methods (getElementByPath, patchReplace, patchSetText, etc.)
- Added explicit cleanup of `optimistic-pending` class in multiple locations

**Impact:**
- Net -123 lines (148 added, 271 removed) in client.js
- Fixes stuck loading cursor bug on optimistic UI updates
- Single source of truth for response handling
- Easier to add features in the future

### 2. Smart IoT Dashboard Demo (Flagship Feature)

A comprehensive demonstration showcasing djust's state management system with 4 coordinated panels:

**Panels:**
1. **Climate Controls** - Temperature/humidity sliders with @optimistic + @throttle + @client_state
2. **Climate Display** - Real-time stats updated via StateBus (heat index, comfort level, energy cost)
3. **Device Management** - Toggleable smart devices with @optimistic updates
4. **Statistics** - Auto-calculated metrics from coordinated state

**Technical Features:**
- @client_state enables cross-panel coordination without JavaScript
- @optimistic provides instant UI feedback (60 FPS slider movement)
- @throttle limits server requests to 10/sec while maintaining real-time updates
- Server-side validation with auto-revert (demonstrates stricter server limits)
- Server-side calculations (heat index, Celsius conversion, comfort level, energy cost)
- Decimal precision support (0.1° increments for temperature)

**Code Quality:**
- ~382 lines of Python (smart_dashboard.py)
- ~568 lines of template (smart_dashboard.html)
- Zero manual JavaScript required
- Excellent inline documentation explaining decorator ordering and trade-offs

### 3. Django App Reorganization

Restructured demo project from monolithic `demo_app` to specialized apps:

- **djust_shared**: Shared components and base classes
- **djust_homepage**: Landing page and navigation
- **djust_demos**: Feature demonstrations (including Smart Dashboard)
- **djust_forms**: Forms demonstrations
- **djust_tests**: Test views for state management
- **djust_docs**: Documentation views

**Impact:**
- Better code organization and discoverability
- Clearer separation of concerns
- Easier to find examples for specific features
- Old demo_app kept for backwards compatibility
