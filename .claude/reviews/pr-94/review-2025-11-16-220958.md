## Summary

This PR implements **Phase 1** of the JIT (Just-In-Time) Auto-Serialization system for djust. It adds a Rust-based template parser that extracts variable access patterns from Django templates, laying the foundation for automatic ORM object serialization. This is a foundational feature that will eliminate the need for manual serialization boilerplate and automatically optimize database queries.

**Type**: Feature (Foundational Infrastructure)
**Impact**: High (enables major framework improvement)
**Changes**: 1,034 additions, 78 deletions across 11 files

---

## What This PR Does

### Core Functionality

1. **Rust Template Variable Extraction**
   - New function: `extract_template_variables(template: &str) -> Result<HashMap<String, Vec<String>>>`
   - Parses Django templates and extracts all variable paths
   - Example: `{{ lease.property.name }}` → `{"lease": ["property.name"]}`
   - Handles all Django template syntax: Variables, If, For, Block, With, Components
   - Performance: <5ms for typical templates

2. **Python Binding via PyO3**
   - Exposed as `djust._rust.extract_template_variables(template)`
   - Returns Python dict with variable paths
   - Proper error handling and type conversion

3. **Comprehensive Testing**
   - 16 new Rust tests (26 total in parser.rs)
   - 34 new Python tests
   - Total: 138 tests covering all use cases
   - All 88 existing tests still pass

4. **Code Quality Improvements**
   - Applied clippy auto-fixes across codebase
   - Removed unused imports
   - Inlined format string arguments
   - Simplified identical if/else blocks

### Strategic Context

This is Phase 1 of a 6-phase project to solve a critical limitation: djust's Rust template engine cannot access nested Django ORM attributes (e.g., `{{ lease.tenant.user.email }}`) because objects are serialized to JSON before crossing the Python/Rust boundary.

**Roadmap**:
- **Phase 1** (this PR): Template variable extraction ✅
- Phase 2: Query Optimizer (3 days)
- Phase 3: Serializer Code Generation (4 days)
- Phase 4: LiveView Integration (2 days)
- Phase 5: Caching Infrastructure (1 day)
- Phase 6: Testing & Documentation (2 days)
