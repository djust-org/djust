## What This PR Does

### Phase 3: Optimistic Updates

Implements the `@optimistic` decorator for instant UI feedback before server validation:

```python
from djust.decorators import optimistic

class TodoView(LiveView):
    @optimistic
    def toggle_item(self, item_id: int, checked: bool = False, **kwargs):
        item = Item.objects.get(id=item_id)
        item.completed = checked
        item.save()
```

**Client-Side Implementation:**
- Heuristic-based updates (checkboxes toggle, inputs update, buttons show loading)
- Automatic revert on server errors with shake animation
- Loading indicators (.optimistic-pending CSS class)
- Server-wins conflict resolution

**Demo Views:**
- `/demos/optimistic-todo/` - Instant checkbox toggles
- `/demos/optimistic-counter/` - Button loading states

### Critical Phase 2 Fix

**Discovered Issue**: The `@debounce` and `@throttle` decorators from PR #40 were **completely non-functional**.

**Root Cause**: Phase 2 decorators were implemented in `python/djust/static/djust/client.js`, which djust never loads. The framework uses embedded JavaScript in `live_view.py`, not external files.

**Impact**: Phase 2 demos didn't work. Users typing in debounced inputs sent events immediately.

**Fix**:
- Ported all Phase 2 decorator logic to embedded JavaScript in `live_view.py`
- Deleted unused `client.js` file (585 lines of dead code)
- Added `window.djustHandleEvent` global API for custom scripts

**Result**:
- @debounce now delays events until typing stops
- @throttle now limits high-frequency events
- All Phase 2 demos work correctly

### 6 Critical Bug Fixes

1. **Empty patches array**: Server sent `{"patches": []}` without html fallback → client hung
2. **JSON double-parsing**: `applyPatches()` parsed already-parsed data → TypeError
3. **Optimistic state clearing**: Button stayed disabled due to timing issue
4. **State reset bug**: `mount()` called on POST, destroying restored state
5. **Patches as string**: Server sent JSON string instead of array
6. **Unused file cleanup**: Removed dead `client.js` code

2. **Empty Patches Fix** (line ~1200):
```python
# OLD (bug): Sent empty array without HTML fallback
if patch_count <= PATCH_THRESHOLD:
    return JsonResponse({"patches": [], "version": version})

# NEW (fixed): Send HTML fallback for empty patches
if patch_count > 0 and patch_count <= PATCH_THRESHOLD:
    return JsonResponse({"patches": patches, "version": version})
else:
    # Empty patches → send HTML
    return JsonResponse({"html": html, "version": version})
```

3. **Optimistic State Clearing** (embedded JS):

clearOptimisticState(eventName);
applyPatches(data.patches);

   - No memory leaks
