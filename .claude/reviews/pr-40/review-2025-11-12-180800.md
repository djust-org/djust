## What This PR Does

Completes the round-trip for `@debounce` and `@throttle` decorators:

### Before (Manual JavaScript - 889 lines)
```javascript
let debounceTimer = null;
searchInput.addEventListener('input', (e) => {
  clearTimeout(debounceTimer);
  debounceTimer = setTimeout(() => {
    sendEvent('search', { query: e.target.value });
  }, 500);
});
```

### After (Pure Python - one line)
```python
@debounce(wait=0.5, max_wait=2.0)
def search(self, query: str = "", **kwargs):
    self.results = Product.objects.filter(name__icontains=query)
```

### Implementation Components

1. **Global State Management** (client.js:35-36)
   - `debounceTimers` Map tracks per-handler debounce state
   - `throttleState` Map tracks per-handler throttle state
   - Independent state prevents handler interference

2. **debounceEvent() Implementation** (client.js:258-291)
   - Classic debounce algorithm: delays until events stop
   - `max_wait` support forces execution after threshold
   - Proper timer cleanup prevents memory leaks
   - Tracks first call time for max_wait enforcement

3. **throttleEvent() Implementation** (client.js:300-370)
   - Leading edge: executes first call immediately
   - Trailing edge: executes last call after interval
   - Both edges: executes first AND last
   - Handles pending data correctly for trailing calls

4. **sendEvent() Pipeline Integration** (client.js:199-230)
   - Checks `window.handlerMetadata` for decorator config
   - Routes to debounce/throttle or immediate send
   - **NEW**: Warns when both decorators present
   - 100% backward compatible (no decorators → immediate send)

5. **Cleanup on Disconnect** (client.js:85-101)
   - **NEW**: Added `cleanupDecoratorState()` method
   - Clears all debounce timers on WebSocket close
   - Clears all throttle timers on WebSocket close
   - Prevents orphaned timers after disconnect

6. **Debug Logging Infrastructure** (client.js:378-393)
   - Category-based filtering
   - Enable: `window.djustDebug = true`
   - Filter: `window.djustDebugCategories = ['debounce', 'throttle']`
   - Zero overhead when disabled

7. **Demo Views for Manual Testing**
   - Debounce demo (`/demos/debounce/`) - interactive search
   - Throttle demo (`/demos/throttle/`) - scroll tracking
   - Both with statistics and debug instructions

**Key Implementation Highlights**:

**1. Cleanup on Disconnect** (lines 85-101):
```javascript
cleanupDecoratorState() {
    // Clear all debounce timers
    this.debounceTimers.forEach(state => {
        if (state.timerId) clearTimeout(state.timerId);
    });
    this.debounceTimers.clear();

    // Clear all throttle timers
    this.throttleState.forEach(state => {
        if (state.timeoutId) clearTimeout(state.timeoutId);
    });
    this.throttleState.clear();
}
```
✅ **Excellent**: Prevents orphaned timers after disconnect

**2. Multiple Decorators Warning** (lines 209-214):
```javascript
if (metadata?.debounce && metadata?.throttle) {

    );
}
```
