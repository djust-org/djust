This PR addresses issue #47 by adding comprehensive documentation for maintaining dual Python/Rust implementations in djust. The new "Dual Implementation Maintenance" section provides clear guidelines, practical examples, and testing strategies for contributors working on the hybrid architecture.

## What This PR Does

Adds a new 169-line section to `CONTRIBUTING.md` covering:

1. **When to Implement in Both Languages**: Clear criteria for dual implementations (UI components, core abstractions, serialization)
2. **Guidelines for Maintaining Consistency**: 5 concrete practices with code examples
3. **Common Patterns**: Real-world examples from StateBackend and LiveView
4. **Testing Strategy**: 4-tier approach (unit, interface, integration, benchmark)
5. **Checklist**: 8-point verification list for dual implementation changes

**Real-World Examples Used**:
- `StateBackend` abstract interface with InMemory/Redis implementations
- `RustLiveView` Python/Rust boundary
- Health check method implementation across backends
- Interface testing with pytest fixtures
