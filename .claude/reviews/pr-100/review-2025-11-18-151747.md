## Summary

This PR implements Phase 3 of the Developer Tooling Plan: an interactive developer debug panel that provides real-time introspection of LiveView event handlers, VDOM patch visualization, and event monitoring. This is a high-quality, production-ready implementation that significantly improves the developer experience for djust users.

## What This PR Does

### Core Features

1. **Server-Side Introspection** (`get_debug_info()` method)
   - Extracts all event handlers with complete parameter signatures, types, and defaults
   - Captures decorator metadata (@debounce, @cache, @throttle, etc.)
   - Lists public instance variables with types and current values
   - Gracefully handles Django's `classonlymethod` decorator edge cases
   - Integrates with Phase 1's validation module (`get_handler_signature_info()`)

2. **Interactive Debug Panel UI** (JavaScript component)
   - Floating button (🔧) in bottom-right corner for easy access
   - Keyboard shortcut: `Ctrl+Shift+D` for power users
   - **4 Information Tabs**:
     - **Event Handlers**: Browse all available handlers with parameters, types, decorators
     - **Event History**: Real-time event log with parameters (last 50 events)
     - **VDOM Patches**: Inspect VDOM patch operations for performance debugging
     - **Variables**: View public instance variables with current values and types
   - Modern dark theme with syntax highlighting
   - Copy-to-clipboard for event JSON (great for test case generation)

3. **Real-Time Monitoring**
   - Hooks into `LiveView.sendEvent()` to capture all outgoing events
   - Hooks into `applyPatches()` to track VDOM updates
   - LRU cache with 50-item limit prevents memory bloat
   - Smooth animations and responsive design

4. **Production Safety**
   - Only active when Django `DEBUG = True`
   - Zero overhead in production (all code gated by DEBUG flag)
   - No debug assets loaded in production builds
   - Safe to deploy without feature flags

### Integration with Previous Phases

- **Phase 1 (Event Parameter Validation)**: Uses `get_handler_signature_info()` to extract parameter metadata
- **Phase 2 (Enhanced @event_handler Decorator)**: Displays decorator metadata stored by `@event_handler`
- Foundation for Phase 4 (Documentation)

## Security Considerations

### Production Safety

**Well-designed security posture**:
1. ✅ Only active when `DEBUG = True` (Django convention for development)
2. ✅ No sensitive data exposure (only exposes what's already in template context)
3. ✅ No authentication bypass or permission escalation
4. ✅ Client-side only (no new server endpoints to secure)
5. ✅ No XSS vulnerabilities (proper JSON escaping in script injection)

### Potential Security Considerations

1. **Debug Mode in Production**
   - **Risk**: If someone accidentally deploys with DEBUG=True, debug panel will be visible
   - **Mitigation**: This is Django's standard practice - DEBUG should never be True in production
   - **Recommendation**: Add note in documentation warning against DEBUG=True in production
   - **Severity**: Low (standard Django security practice)

2. **Information Disclosure**
   - **Risk**: Debug panel exposes handler signatures, variable names, and values
   - **Mitigation**: Only visible when DEBUG=True, which exposes far more sensitive data anyway
   - **Recommendation**: No additional mitigation needed
   - **Severity**: Very Low (consistent with Django's DEBUG mode behavior)

### XSS Analysis

**Injection points analyzed**:
1. `window.DJUST_DEBUG_INFO = {json.dumps(debug_info)}` - ✅ Safe (json.dumps escapes properly)
2. HTML rendering in JavaScript - ✅ Safe (values from server, not user input)
3. Event parameter display - ⚠️ Potential issue if user-controlled data contains malicious HTML

**Recommendation**: Add HTML escaping in `renderEventHistory()` when displaying `event.params`:
```javascript
// Before:
<pre>${JSON.stringify(event.params, null, 2)}</pre>

// After (safer):
<pre>${this.escapeHtml(JSON.stringify(event.params, null, 2))}</pre>
```

**Severity**: Low (only affects developer's own browser in DEBUG mode, not production users)

**Verdict**: Security is good, with one minor enhancement recommended for defense-in-depth.
